Less obvious perks of using 'live' data resources include logins and payments. How could Twitter be used in these two scenarios?

  1. Regarding Payments: One could use Twitter to send an automatic tweet to followers once a purchase has been made. It could also just be given as an option upon a successful checkout.

  2. Regarding Logging in: You could allow a user to authenticate into your service or app using their twitter credentials. This is convenient to people who use Twitter because it means that they don't have to create another user account to keep track of.

Visit Programmableweb.com and search for JSON-P data sources. Why would we need JSON-P, specifically?

  • Due to the Same Origin Policy, normal JSON data requests that are made to a server outside of the server the application is running on is denied. JSON-P allows us to retrieve JSON from outside sources using the <script> tag.

    • Based on the JSON-P search results: what data resource(s) do you think might pair well with which types of Native device features?

    1. Camera - Flickr, Instagram
    2. Geolocation - Foursquare, Bandsintown
    3. Notifications - Etsy, Facebook, Twitter
    4. Media - Vimeo
    5. Contacts - LinkedIn

    Please describe any design and functionality intentions you may have for your Demo app components this month.

  • I'm considering using an API called "Bandsintown." This API allows users to view local concert schedules and view live music recommendations. I think it'd be a great mash-up with Twitter. You could use it to locate local shows around town and then subscribe to the twitter feeds of the bands you're interested in seeing. This could be a fun way to get to know the band a little before actually going to see them live. Who knows, it may even give you some things to talk to the band about.

    • Briefly describe some past 'lessons learned' regarding workflow as it relates to research, design, development, etc.

  • I've got a habit of wanting things to look nice right out of the gate. I'm learning that I need to get the base functionality working first before I start to concern myself with the apps aesthetics. I've also learned to do my research and design ahead of time. It's so much easier to implement an idea when I know what it is I'm going for and how exactly I'd like to accomplish it.
