Less obvious perks of using 'live' data resources include logins and payments. How could Twitter be used in these two scenarios?

  1. Regarding Payments: One could use Twitter to send an automatic tweet to followers once a purchase has been made. It could also just be given as an option upon a successful checkout.

  2. Regarding Logging in: You could allow a user to authenticate into your service or app using their twitter credentials. This is convenient to people who use Twitter because it means that they don't have to create another user account to keep track of.

Visit Programmableweb.com and search for JSON-P data sources. Why would we need JSON-P, specifically?

  • Due to the Same Origin Policy, normal JSON data requests that are made to a server outside of the server the application is running on is denied. JSON-P allows us to retrieve JSON from outside sources using the <script> tag.

    Some of the ways that I could improve my HTML to make it more accessible:

    Carefully planned and consistently structured semantic mark-up. It would be beneficial to establish a consistent work flow that utilized the various mark up tools in HTML5.


    Some of the common frustrations have to do with the organization of the content on the page. As developers or designers, we tend to take for granted our ability to see the page and trade logical structure for more visually appealing layouts. This can lead to confusion for visitors using screen readers.


    I decided to conduct a little research. I was curious to see if it were possible to detect when a user is using a screen reader on your site. I was thinking that one could dynamically adjust the layout of the content based on a simple Boolean: screeenReader === true; What I discovered is that it’s a little more complicated than that. As it stands there really isn’t a way to do this consistently, and it appears that it may even be a violation of privacy since it would disclose whether or not a visitor to your site had a disability. That said, I really appreciate this assignment. Accessibility wasn’t really something I was thinking much about, but in hind site it’s absolutely crucial, AND the right thing to do.


    I know I’m not going to be alone in my first selection, but Apple’s VoiceOver software is terrific. Not only is it easy to use but it’s really robust. I actually took some time to go through the VoiceOver guided tutorial and was really impressed with what I found. Some features include the “Track Pad Commander” which allows a user to control the screen reader using multitouch gestures on the trackpad. It also comes preinstalled with over 70 drivers for various refreshable braille displays so users can simply plug and play.


    Another device that I thought was really awesome is called “Access4Kids.” It’s an assistive input device that provides children with limited upper-body motor control, access to rehabilitation applications via touch screen devices. It uses three sensors that collect a series of inputs that the children are capable of doing, and converts them into the standard multitouch inputs we all take for granted.


    There are some really terrific applications out there that are designed exclusively for accessibility. The two that I chose to write about are “BrailleTouch” and “soundAMP.”


    BrailleTouch allows users to type in braille using their touch screen phones. The phone is held with the screen facing away and the user positions their fingers in braille patterns along the edges of the screen to type.


    soundAMP converts a users smart phone into a listening device. Users can amplify their surroundings using their own earbuds and smartphone.
